# REASONIX.md ‚Äî mic-check / Freely Spoken

## Stack

- **Mobile**: React Native 0.81, Expo SDK 54 (new architecture), TypeScript (strict)
- **Navigation**: expo-router (file-based, in `app/`)
- **Audio**: expo-av for recording, expo-speech-recognition for on-device transcription
- **On-device AI**: `@ratley/react-native-apple-foundation-models` ‚Äî sentiment + anonymization via Apple Foundation Models
- **Backend**: FastAPI with uvicorn, httpx, pydantic ‚Äî deployed to Fly.io
- **Tooling**: eslint (expo-config flat config), vitest (tests in `hooks/__tests__/`), no formatter

## Layout

| Path | Contents |
|------|----------|
| `app/` | expo-router screens (`index.tsx` = main UI, `debug.tsx`, `_layout.tsx`) |
| `hooks/` | Pipeline hooks + `sentiment-utils.ts` (pure parsing functions, RN-free) |
| `components/` | Shared UI (`themed-text.tsx`, `themed-view.tsx`) |
| `constants/` | Brand config (`brand.ts`), theme (`theme.ts`) |
| `services/` | `lookup-client.ts` ‚Äî the only network egress |
| `server/` | FastAPI backend (`app.main`, `app.lookup.*`, `app.providers.*`) |
| `tools/` | Dev harnesses (`sentiment-cli/`, `lookup-harness/`) |
| `docs/` | Plans, architecture reviews, debug/testing notes |

## Commands

```sh
npm start          # expo start
npm run ios        # expo run:ios (connected device only ‚Äî no simulator)
npm run lint       # expo lint (eslint via expo-config)
# Server (cd server/):
pip install -e .   # install FastAPI + dependencies
uvicorn app.main:app --reload --port 8080
```

Tests are in `hooks/__tests__/` (vitest, 33+ tests on sentiment parsing). CI runs `npm run lint` + `npm run typecheck` + `npm test` on every push/PR to main.

## Conventions

- **Named exports everywhere** ‚Äî hooks and components use `export function useX()` / `export function X()`, imported as `import { X } from '@/path'`
- **Path alias** `@/` maps to project root (`tsconfig.json` paths)
- **Hook state pattern**: each pipeline hook returns `{ result, isX, error, run, reset }` ‚Äî wired together by `useEffect` chains in `app/index.tsx`
- **Server prompt iteration**: prompts and adapters live in both `server/` (production) and `tools/lookup-harness/` (iteration); changes should port to both

## Watch out for

- **No Expo Go** ‚Äî native modules (`expo-av`, `expo-speech-recognition`, `@ratley/react-native-apple-foundation-models`) require `expo run:ios --device`
- **Hardware requirement** ‚Äî iPhone 15 Pro+ / 16 series, iOS 26+, Apple Intelligence enabled in Settings
- **`/ios` and `/android` are gitignored** ‚Äî regenerated by `npx expo prebuild`. Never edit native files directly
- **Server is a separate service** ‚Äî deployed to Fly.io, not part of the mobile app build
